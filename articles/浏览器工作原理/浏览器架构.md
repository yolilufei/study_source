### 为什么是 Chrome 浏览器
因为 Chrome、微软的 Edge 以及国内的大部分主流浏览器，都是基于 Chromium 二次开发而来；而 Chrome 是 Google 的官方发行版，特性和 Chromium 基本一样，只存在一些产品层面差异；再加上 Chrome 是目前世界上使用率最高的浏览器，所以 Chrome 最具代表性。
### 提前要了解的概念—— CPU、GPU、Memory 和 multi-process
受限于文章重点，这些概念只做简要描述。
#### CPU(Central Processing Unit)
CPU是中央处理器，可以看作是计算机的大脑。一个 CPU 核心，就像图中一样，可以处理大量的任务。得益于硬件的发展，计算机由原先的单核CPU发展为多核CPU，大大提高了计算机的性能。  
想了解更多有关CPU的概念:[维基百科](https://zh.wikipedia.org/wiki/%E4%B8%AD%E5%A4%AE%E5%A4%84%E7%90%86%E5%99%A8)

![CPU][1]
CPU 核心就像办公室职员一样坐在各自的位置处理进来的任务


#### GPU(Graphics Processing Unit)
GPU是图形处理器。顾名思义，对图像的处理很在行。GPU的特点就是非常适合处理海量的逻辑简单的任务，比如游戏中对每一个顶点进行同样的坐标变换，对每一个顶点按照同样的光照模型计算颜色值等。  
更多 CPU 和 GPU 的区别，可以参考[这里](https://www.zhihu.com/question/19903344)

#### 进程和线程
概述：进程和线程都是一个时间段的描述，是CPU工作时间段的描述。是运行中的程序指令的一种描述，这需要与程序中的代码区别开来。 
进程可以简单理解为：一个进程就是一个应用实例(单进程)。启动一个程序的时候，操作系统会为该程序创建一块内存，用来存放代码、运行中的数据和一个执行任务的主线程，我们把这样的一个运行环境叫进程
线程负责执行应用运行时的不同工作。 
这里有更生动的解释：[进程和线程](https://www.ruanyifeng.com/blog/2013/04/processes_and_threads.html)

### 浏览器架构
浏览器是一个应用，基于对进程和线程的理解，浏览器可以是**单进程多线程**，也可以是**多进程多线程**。
![单进程多线程][2]
<p style="text-align:center">左：单进程多线程架构 右：多进程多线程架构</p>

#### 单进程架构
![单进程架构][3]
单进程架构只有一个浏览器进程，所有的功能模块都以线程的形式运行在该进程中，包括渲染线程、网络线程等。使用这种架构的浏览器都面临以下三个问题：
- 不稳定。由于线程出错会影响进程的执行，甚至会导致崩溃。而插件和渲染线程就是其中不稳定线程的代表（早期的音视频都是依靠插件才能正确运行的，像我们最常见的 flash 插件）。有问题的插件或者有问题的代码（死循环、内存泄漏）都会导致浏览器进程的崩溃，从而导致浏览器不稳定。
- 不安全。 浏览器拥有操作系统的权限，可以读写文件。如果插件或者js也获取了这个权限，那么插件或js脚本也可以读写文件。我们的计算机相当于完全暴露，毫无隐私可言。
- 不流畅。在单线程架构中，渲染线程包括JS执行环境、插件模块、渲染模块等。意味着同一时刻只能有一个模块运行。如果js脚本很复杂，js解释执行的时间就会变长，这样就会导致页面卡顿、无响应。同时由于复杂页面导致内存回收不完全，随着浏览器运行时间加长，内存占用越来越大，页面的响应也会越来越慢。

针对以上问题，衍生出了多进程模式。
